## 2. プロダクト概要

### 2.1 プロジェクト名

**Kokeniwa OS(苔庭OS)**

---

### 2.2 プロダクトコンセプト

Kokeniwa OS(苔庭OS)は、個人PCや自前サーバー上で起動し、ブラウザから利用できるOSSのWebデスクトップ型ワークスペース基盤である。

OSそのものを開発するのではなく、OSのような見た目・操作性・アプリ管理機能をWebアプリケーションとして提供する。

ユーザーは、Webアプリ、Dockerアプリ、VNC/リモートデスクトップ、ローカルファイル、AIアシスタント、業務用ダッシュボードなどを1つの統合UIから利用できる。

---

### 2.3 一文での説明

個人PCではじめられ、必要に応じて自前サーバーへ移行でき、PC・iPad・店舗端末から利用できる、ローカルファーストなOSS Web Workspace。

---

### 2.4 解決したい課題

現在、個人開発者、研究室、小規模企業、店舗では、複数のWebアプリやローカルツールを個別に管理していることが多い。

例：

* Open WebUI
* Ollama
* n8n
* JupyterLab
* VS Code Server
* File Browser
* 社内Webアプリ
* 店舗用予約管理画面
* CSV/PDF分析ツール
* VNCデスクトップ
* Docker管理画面

これらは便利だが、以下の問題がある。

1. URLやポートが分散しており、使いづらい
2. Dockerアプリの起動・停止・更新が非エンジニアには難しい
3. iPadや店舗端末から安全に使うためのUIが不足している
4. 個人PCからサーバー運用へ移行しづらい
5. 外部SaaSに依存せず、自分の環境で管理したい需要がある
6. AIツール、ファイル、業務画面、実行環境が分断されている
7. 小規模組織向けに、軽量で自由にカスタマイズできる業務ポータルが少ない

Kokeniwa OS(苔庭OS)は、これらを1つのWebデスクトップ環境に統合する。

---

## 3. プロダクトの基本方針

### 3.1 Local-first

基本方針はローカルファーストとする。

* ユーザーのPC上で起動できる
* データは原則としてユーザーの管理下に保存する
* 外部SaaSに依存しない
* 自前サーバーへ移行可能にする
* インターネット接続がなくてもできるだけ画面表示・ローカル利用ができる

---

### 3.2 Self-hostable

組織利用では、自前のLinuxサーバー、ミニPC、店舗PC、NAS、研究室サーバー等にデプロイできることを重視する。

* Docker Composeで導入できる
* LAN内のPC、iPad、タブレットからアクセスできる
* データベース、ファイル、アプリ設定を自組織内に保持できる
* 将来的にバックアップ・リストアにより別サーバーへ移行できる

---

### 3.3 Browser-based

利用者は専用クライアントを必須とせず、ブラウザから利用できる。

* PCブラウザ対応
* iPad Safari / Chrome対応
* Androidタブレット対応
* PWAとしてホーム画面追加可能
* キオスク端末・店舗端末向けのフルスクリーンUI対応

---

### 3.4 Customizable

ユーザーが自分の用途に合わせてワークスペースを変更できることを重視する。

* アプリ追加
* レイアウト変更
* アイコン配置
* ワークスペース切り替え
* スタッフ用画面、管理者用画面、キオスク画面の分離
* app manifestによるアプリ定義
* 設定・レイアウトのエクスポート/インポート

---

### 3.5 OSS-first

本体はOSSとして公開する。

* 個人・研究室・小規模企業が無料で試せる
* コントリビューターがアプリテンプレートを追加できる
* セルフホスト利用を前提にする
* 将来的に有償化する場合も、コア機能はOSSとして維持する

---

## 7. 機能要件

## 7.1 認証・ユーザー管理

### 7.1.1 ログイン

ユーザーはログインしてWorkspaceを利用できる。

#### 要件

* メールアドレスまたはユーザー名でログインできる
* パスワード認証に対応する
* 初回起動時にOwnerユーザーを作成できる
* Personal Modeでは簡易ログインなしモードも選択可能にする
* Server Modeではログインを必須にする

#### 優先度

* MVP: 必須

---

### 7.1.2 ロール管理

以下のロールを定義する。

| ロール       | 説明                       |
| --------- | ------------------------ |
| owner     | システム全体の所有者。全操作可能         |
| admin     | アプリ、ユーザー、レイアウトを管理可能      |
| staff     | 指定されたアプリ・画面のみ利用可能        |
| viewer    | 閲覧のみ可能                   |
| kiosk     | 固定画面のみ利用可能               |
| developer | app manifestやテンプレート追加が可能 |

#### 優先度

* MVP: owner/admin/staffのみ
* v0.3以降: viewer/kiosk/developer追加

---

### 7.1.3 権限管理

操作単位で権限を管理する。

#### 権限例

* アプリ起動
* アプリ追加
* アプリ削除
* Docker起動/停止
* レイアウト編集
* ユーザー追加
* ファイル閲覧
* ファイル削除
* ログ閲覧
* バックアップ作成
* AI実行
* 危険操作承認

## 7.4 App Manifest / App Registry

### 7.4.1 App Manifest

すべてのアプリはmanifestで定義する。

#### manifest例

```yaml
id: open-webui
name: Open WebUI
type: docker
category: ai
icon: open-webui.svg
description: Local AI chat interface
entry: /apps/open-webui
permissions:
  - network
  - storage
  - ai
```

---

### 7.4.2 アプリ種別

以下のアプリ種別をサポートする。

| 種別            | 説明                          | MVP対応 |
| ------------- | --------------------------- | ----- |
| iframe        | iframeで表示するWebアプリ           | 対応    |
| external_link | 新規タブまたは外部リンクで開くアプリ          | 対応    |
| docker        | Docker Composeで起動するアプリ      | v0.2  |
| vnc           | VNC/noVNC/Guacamoleで表示するアプリ | v0.3  |
| file          | ファイルビューア/フォルダ               | v0.2  |
| ai_tool       | AI機能を持つ内部アプリ                | v0.4  |
| custom        | 独自プラグイン/拡張                  | 将来    |

Docker操作はBackend経由でのみ行い、FrontendからDocker socketへ直接アクセスしない。

* rootless Docker対応を検討

* secretsを平文表示しない

#### 対策

* 自前DockerアプリはReverse Proxyで統合する

#### 対策

* Audit Log
* 操作前確認
* rootless Docker対応の検討

### 18.4 店舗・企業利用に必要な信頼性不足

## 20. 最終的なプロダクト定義

Kokeniwa OS(苔庭OS)は、OSそのものではなく、ブラウザ上でOSのような操作性を提供するWeb Workspace基盤である。

個人PCで簡単に始められ、必要に応じて自前サーバーに移行でき、iPadや店舗端末からも利用できる。

コア価値は、Webアプリ、Dockerアプリ、VNC環境、AI機能、ローカルファイルを、ユーザーごとのカスタムWorkspaceとして安全に統合することである。

最初のMVPでは、WebデスクトップUI、App Manifest、アプリランチャー、ウィンドウ管理、レイアウト保存に集中する。

その後、Docker App Manager、File Center、Backup/Restore、Kiosk Mode、VNC、AI Assistantを段階的に追加する。

この順序で開発することで、個人開発者・研究室・小規模企業・店舗に対して、現実的に導入可能なOSSプロダクトとして成長させる。
